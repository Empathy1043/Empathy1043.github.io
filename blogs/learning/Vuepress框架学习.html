# VuePress框架学习

起因是因为看到一位师兄的博客是通过VuePress来搭建完成的，界面简洁美观。尝试学习使用时候才发现有太多的坑了😂

# 什么是Vuepress?

VuePress 是一个以 Markdown 为中心的静态网站生成器。你可以使用 Markdown (opens new window) 来书写内容（如文档、博客等），然后 VuePress 会帮助你生成一个静态网站来展示它。

Vuepress是基于markdown来生成网页界面的，不需要繁琐的前端界面设计和相关样式的设置。这也是我为什么使用它的原因，我主要是想使用它来进行内容的输出，使用markdown来书写的成本降低了不少。

# 关键词

  • 轻量化
  • 支持插件系统
  • 基于markdown渲染

	  configureWebpack: {

	        '@assets': path.resolve(__dirname, '../assets')

	  },
	
	  // Markdown 配置

# 5.基于Github Pages的网页部署

什么是 Github Pages 呢？简单说就是 Github 提供的、用于搭建个人网站的静态站点托管服务。很多人用它搭建个人博客。这种方式的好处是免费、方便，坏处是速度可能会有些慢、不能被国内的搜索引擎收录。

  1. 新建仓库一： USERNAME.github.io （不用克隆到本地） 注意：USERNAME 必须是你 Github 的账号名称
  2. 新建仓库二：随便起一个名字，比如：vuepressBlog （克隆到本地）

    • 使用工具包的，将vuepress-starter中的内容拷贝到vuepressBlog文件夹中

    • 自己从头搭建的，将 vuepress-starter文件夹的内容拷贝到仓库二，并在根目录下创建 deploy.sh 文件，内容如下**（注意替换USERNAME）**：

      #!/usr/bin/env sh
# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.yourwebsite.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果你想要部署到 https://USERNAME.github.io
git push -f git@github.com:USERNAME/USERNAME.github.io.git master

# 如果发布到 https://USERNAME.github.io/<REPO>  REPO=github上的项目
# git push -f git@github.com:USERNAME/<REPO>.git master:gh-pages

cd -
